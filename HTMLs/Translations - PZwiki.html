This article is about translating the game or a mod. For a guide to translating the wiki, see Help:Translations.

Official translations used to be done on ProjectZomboidTranslations GitHub, however starting with Build 42 a professional studio was hired for translations.

Information about the game's translation system. Translations need to be saved in the encoding of the language used by the game. Files also need to follow the naming conventions and format of the translation types.

Translation types

Commonly used files

These files follow the same naming convention Translate/{language}/{File}_{language}.txt and use the same format.

List of translation files
File Entries prefix Function Notes

ContextMenu ContextMenu_ getText ISContextMenu - in-game context menus.
DynamicRadio AEBS_ (optional) getRadioText Dynamic radio, e.g. weather and helicopter.

EvolvedRecipeName EvolvedRecipeName_ (optional) Translator.getItemEvolvedRecipeName Ingredient names due for change for evolved recipe UI ("Can of tomatoes" becomes "Tomato").
Farming Farming_ getText Farming menus.
Fluids Fluid_Name_ getText Fluids.
GameSound GameSound_ getText Sound categories for options, debuging and descriptions of sounds.
IG_UI IGUI_ getText In-game UI elements.
ItemName ItemName_ (optional) getItemNameFromFullType Item display names.
MakeUp MakeUpType_ or MakeUpCategory_ getText Make up names.
MapLabel MapLabel_ getText Region names.
Moodles Moodles_ getText Moodles descriptions.
Moveables Translator.getMoveableDisplayName Names of tiles - sprites.
MultiStageBuild MultiStageBuild_ Translator.getMultiStageBuild Automatically appends the prefix when you get the translation.
Print_Media Print_Media_ getText Flier picture.
Print_Text Print_Text_ getText Flier failback text of content and description.
RadioData RD_ getText Radio media content. Key is prefix + GUID.
Recipes Recipe_ (optional) getRecipeDisplayName Crafting recipe names.
RecipeGroups RecipeGroup_ Translator.getRecipeGroupName Crafting recipe groups.
Recorded_Media RM_ getText Television media content. Key is prefix + GUID.
Sandbox Sandbox_ getText Sandbox options names, enumerate choices and tooltips. Sandbox option categories and buttons.
Stash Stash_ getText Map annotations.
SurvivalGuide SurvivalGuide_ getText Tutorial guides.
SurvivorNames SurvivorSurname_ or SurvivorName_ getText Random character names.
Tooltip Tooltip_ getText
UI UI_ getText Main menu and option UI elements.
  • Prefixes can be handled differently; they can be optional, removed from key maps or not used at all.
  • Use the full type where type is used as key.
  • Moveables use Group Name Custom Name for names and keys. These characters , -, ', . are converted.

Examples

Example of ContextMenu: Source: media\lua\shared\Translate\ContextMenu_EN.txt 

ContextMenu_EN = {
    ContextMenu_Destroy = "Destroy ",
}

Example of text split in multiple lines. 

SurvivalGuide_EN = {
    SurvivalGuide_entrie11moreinfo = " <H1> FORAGING / SEARCH SYSTEM <BR> " ..
                                     " ..." ..
                                     " For example, trash can be found in trash piles, chipped stones are often found in cracks and potholes on roads. <LINE> <LINE> ",
}
The translation parser skips the first line, then breaks the line into key - text format. Required characters: = and ". 
line 1: skipped
key = "text",

Recipes

This section may be outdated.
Editors are encouraged to update this section with new information. Edit (Create account)

Script parser uses `replace("recipe","")` to make original name, sets translated recipe name. Translator populates recipe maps (normal and 'dot' maps) and uses *replace("Recipe_","")*, *replace("_"," ")* for the keys. 

Examples A, both translation lines work.

script: 'recipe Convert A B {...}'
original name: 'Convert A B'
translation line: 'Recipe_Convert_A_B = "Convert Item to Weapon"'
translation line: 'Convert A B = "Convert Item to Weapon"'
translation key: 'Convert A B'

Example B, no match for translation, do not use `_` in recipe scripts

script: 'recipe Convert_A_B'
original name: 'Convert_A_B'
translation key: 'Convert A B'

Map information

Map translations use the following files: 

media/
 maps/
    <map folder>/
        map.info
        ...
 lua/
     shared/
         Translate/
             <language code>/
                 <map folder>/
                     description.txt
                     title.txt
  • map.info - contains informations about the map.
  • description.txt - description of the map.
  • title.txt - title of the map.

Example Riverside description: Source: Translate\EN\Riverside, KY\description.txt

<CENTRE> <SIZE:medium> RIVERSIDE <LINE> <LINE>

Radio data

This section may be outdated.
Editors are encouraged to update this section with new information. Edit (Create account)

Radio Data is read from media/radio/. Radio sources are xml files, translations are txt files. The recommended way to translate these can be found on the translation GitHub, however you can also edit them just like any other txt file.

Languages

The game uses languages added to the media/lua/shared/Translate folder, information about the language is stored in the language.txt file of each language directory.

For Build 41, most language use different encodings. But for Build 42, almost all languages use the UTF-8 encoding, except for Czech (CS) which uses cp1250, Catalan (CA) which uses ISO-8859-15, and Argentinian Spanish (AR) which uses cp1252.
